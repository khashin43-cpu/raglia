# ЛИА 0.2

ЛИА — корпоративный интерфейс для работы с ИИ, пользовательскими ассистентами
по базам знаний RAGFlow 0.24 и офисными документами.

## Возможности

- системный помощник **Ария** через OpenAI-совместимый API;
- **Консультант по ЛНАД**;
- пользовательские ассистенты с подключёнными пространствами и базами RAGFlow;
- управление пространствами, базами, документами, участниками и общими ассистентами;
- выбор одного или нескольких ассистентов рядом с кнопкой `+` в чате;
- ссылки на использованные документы с указанием пространства и базы;
- просмотр оригинала документа через защищённый маршрут ЛИА;
- создание и редактирование DOCX, XLSX и PPTX через OfficeCLI;
- предпросмотр офисных документов и скачивание результата прямо в чате;
- сохранение изменённого документа новой версией в RAGFlow с запуском индексации;
- загрузка истории чата в базу знаний в формате Markdown;
- совместимость с моделями, знаниями, промптами, навыками и инструментами исходной платформы.

## 1. Системные требования

Поддерживаются macOS и Linux.

Понадобятся:

- Git;
- `curl`;
- Python 3.11 или 3.12;
- Node.js 20 или 22;
- npm;
- `uv` для Python-зависимостей.

Node.js 23 и новее не рекомендуется: проект объявляет поддержку до Node.js 22.

Проверка:

```bash
git --version
python3 --version
node --version
npm --version
```

### Установка uv

Если `uv --version` не выполняется:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

Откройте новый терминал или временно добавьте установленную программу в `PATH`:

```bash
export PATH="$HOME/.local/bin:$PATH"
uv --version
```

## 2. Клонирование и установка

```bash
git clone https://github.com/khashin43-cpu/raglia.git
cd raglia
```

Создайте виртуальное окружение и установите зафиксированные backend-зависимости:

```bash
uv sync --frozen
```

Команду нужно выполнять из корня проекта, а не из каталога `backend`.

Установите frontend-зависимости:

```bash
npm ci
```

Если `npm ci` сообщает о несовместимой версии Node.js, переключитесь на Node.js 20 или 22.

## 3. Переменные окружения

Создайте локальный файл `.env` из шаблона:

```bash
cp .env.example .env
```

Сгенерируйте секрет приложения:

```bash
openssl rand -hex 32
```

Вставьте результат в `WEBUI_SECRET_KEY` и заполните интеграции:

```env
WEBUI_NAME='ЛИА'
WEBUI_SECRET_KEY='длинная-случайная-строка'

# Локальные модели Ollama не используются в стандартной конфигурации ЛИА.
ENABLE_OLLAMA_API='false'

# RAGFlow 0.24. Адрес указывается без /api/v1.
RAGFLOW_URL='http://ragflow.example:80'
RAGFLOW_API_KEY='ключ-ragflow'

# Диагностический адрес MinIO. Доступ к оригиналам выполняется через API RAGFlow,
# поэтому логин и пароль MinIO приложению ЛИА не требуются.
RAGFLOW_MINIO_URL='http://minio.example:9001'

# Основная системная модель Ария.
LIA_ARIA_ENABLED='true'
LIA_ARIA_API_BASE_URL='https://провайдер.example/v1/chat/completions'
LIA_ARIA_API_KEY='ключ-провайдера'
LIA_ARIA_MODEL='имя-модели'

# Офисный ассистент и редактирование документов пользовательскими ассистентами.
LIA_OFFICE_ENABLED='true'
LIA_OFFICECLI_BINARY='./backend/bin/officecli'
LIA_OFFICECLI_WORK_DIR='./backend/data/officecli'
LIA_OFFICECLI_TIMEOUT='120'

# Консультант по ЛНАД. Можно отключить, если сервис не используется.
LIA_LND_ENABLED='true'
LIA_LND_API_BASE_URL='http://адрес-сервиса:7777/v1'
LIA_LND_API_KEY=''

CORS_ALLOW_ORIGIN='*'
FORWARDED_ALLOW_IPS='*'
```

Правила адресов:

- `RAGFLOW_URL` — адрес сервера RAGFlow без `/api/v1`;
- `RAGFLOW_MINIO_URL` — адрес S3 API или проксируемого сервиса MinIO в вашей установке;
- `LIA_ARIA_API_BASE_URL` и `LIA_LND_API_BASE_URL` могут указывать на корень `/v1`
  либо непосредственно на `/chat/completions`;
- секреты нельзя добавлять в `.env.example` или исходный код.

Файл `.env` исключён из Git.

## 4. Получение API-ключа RAGFlow

1. Откройте интерфейс RAGFlow.
2. Войдите под пользователем, которому доступны нужные базы.
3. Откройте настройки аккаунта и раздел API keys.
4. Создайте ключ и вставьте его в `RAGFLOW_API_KEY`.

ЛИА использует публичный API RAGFlow 0.24 для пространств, баз, документов,
поиска, скачивания оригиналов и запуска парсинга. Доступ пользователя дополнительно
проверяется по участию в пространстве ЛИА.

## 5. Установка OfficeCLI

Бинарный файл OfficeCLI не хранится в Git. Установите его после клонирования:

```bash
./backend/install_officecli.sh
```

Скрипт:

- выбирает бинарник для macOS/Linux и архитектуры компьютера;
- проверяет SHA-256;
- помещает программу в `backend/bin/officecli`;
- не требует установленного Microsoft Office.

Проверка:

```bash
./backend/bin/officecli --version
```

## 6. Запуск для разработки

Нужны два терминала.

### Терминал 1 — backend

```bash
cd raglia
BACKEND_RELOAD=true ./backend/run_backend.sh
```

Backend будет доступен по адресу <http://127.0.0.1:8080>.

При запуске скрипт выводит активные адреса RAGFlow, MinIO, Арии, OfficeCLI и ЛНАД,
но не выводит API-ключи.

### Терминал 2 — frontend

```bash
cd raglia
npm run dev
```

Frontend будет доступен по адресу <http://localhost:5173>.

При первом запуске откройте frontend и создайте пользователя. Первый пользователь
чистой установки получает права администратора.

## 7. Доступ с другого компьютера в локальной сети

Запустите backend на всех сетевых интерфейсах:

```bash
HOST=0.0.0.0 BACKEND_RELOAD=true ./backend/run_backend.sh
```

Frontend-команда уже использует сетевой режим Vite. Если нужен явный запуск:

```bash
npm run dev -- --host 0.0.0.0
```

Узнайте адрес компьютера.

macOS:

```bash
ipconfig getifaddr en0
```

Linux:

```bash
hostname -I
```

На другом компьютере откройте:

```text
http://IP-КОМПЬЮТЕРА:5173
```

Порты 5173 и 8080 должны быть разрешены локальным firewall. Для постоянной эксплуатации
рекомендуется использовать reverse proxy и HTTPS, а не публиковать dev-сервер в интернет.

## 8. Production-сборка

Сборка требует доступа к интернету для получения пакетов Pyodide:

```bash
npm run build
```

После успешной сборки backend может обслуживать готовый frontend:

```bash
FRONTEND_DEV=false HOST=0.0.0.0 ./backend/run_backend.sh
```

Откройте <http://localhost:8080> или адрес сервера на порту 8080.

Для production обязательно:

- задайте уникальный `WEBUI_SECRET_KEY`;
- ограничьте `CORS_ALLOW_ORIGIN` нужным доменом;
- используйте HTTPS;
- не публикуйте `.env`;
- ограничьте сетевой доступ к RAGFlow, MinIO и внутреннему сервису ЛНАД.

## 9. Проверка интеграций

После входа:

1. Откройте **Ассистенты по Вашим базам знаний**.
2. Проверьте статус соединения с RAGFlow.
3. Создайте пространство и базу либо используйте уже доступные.
4. Загрузите документ и дождитесь завершения индексации.
5. Создайте ассистента и подключите пространство.
6. Выберите ассистента рядом с кнопкой `+` в строке ввода.
7. Задайте вопрос по документу и откройте показанный источник.
8. Нажмите **Изменить в чате**, опишите правки и проверьте предпросмотр.
9. Для добавления результата в RAGFlow явно попросите сохранить документ в базу.

Исходный файл не перезаписывается автоматически: изменённый документ загружается новой
версией, после чего RAGFlow запускает его индексацию.

## 10. Тесты

Backend-тесты интеграций:

```bash
uv run pytest -q backend/tests
```

Проверка production-сборки без повторной подготовки Pyodide:

```bash
./node_modules/.bin/vite build
```

Полная сборка:

```bash
npm run build
```

## 11. Локальные данные и резервное копирование

В репозиторий намеренно не входят:

- `.env` и API-ключи;
- `.venv`;
- `node_modules`;
- `build` и `.svelte-kit`;
- `backend/data` с пользователями, чатами, загруженными документами и индексами;
- `backend/bin/officecli`;
- рабочие документы OfficeCLI;
- логи и кеши.

Чтобы перенести не только код, но и состояние конкретной установки, отдельно и безопасно
скопируйте `.env` и `backend/data`. Остановите backend перед копированием SQLite-файлов.

## 12. Частые ошибки

### `uv: command not found`

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"
```

### `WEBUI_SECRET_KEY is not set`

Создайте `.env` и задайте длинное случайное значение `WEBUI_SECRET_KEY`.

### `RAGFLOW_API_KEY is required when RAGFLOW_URL is set`

Заполните оба параметра либо оставьте оба пустыми, если интеграция отключена.

### Ошибки соединения с `localhost:11434`

Если Ollama не используется, задайте:

```env
ENABLE_OLLAMA_API='false'
```

### OfficeCLI не найден

```bash
./backend/install_officecli.sh
```

### Порт занят

macOS/Linux:

```bash
lsof -nP -iTCP:8080 -sTCP:LISTEN
lsof -nP -iTCP:5173 -sTCP:LISTEN
```

Остановите старый процесс сочетанием `Ctrl+C` в его терминале.
